### Project Workflow

This project follows a structured data analytics pipeline, starting from business problem identification to final insight generation and reporting.

### 1. Business Problem Definition

The project begins with identifying and clearly defining the business problem. This step establishes the objectives of the analysis and determines the key questions that the data analysis aims to answer.

### 2. Data Ingestion

Raw datasets were ingested into a relational database. This process involved creating database tables and loading the raw data into the database to enable efficient querying and processing.

### 3. SQL Exploratory Data Analysis (EDA)

Initial exploration of the data was conducted using SQL queries. This included examining data distributions, checking for inconsistencies, and understanding relationships between tables.

### 4. Data Transformation
   
Data from multiple tables was transformed using advanced SQL techniques including:
JOIN operations
Common Table Expressions (CTEs)
Aggregations
Filtering and grouping
These operations were used to combine and structure the data into a meaningful analytical format.

### 5. Final Analytical Table Creation

After performing the necessary transformations, a final analytical dataset was generated. This table contains the cleaned and aggregated data required for further statistical and analytical processing.

### 6. Data Export

The final analytical table was exported from the database as a CSV file, making it suitable for further analysis using Python-based data analysis tools.

### 7. Python-Based Data Analysis

The exported dataset was analyzed using Python and data analysis libraries such as Pandas. This stage involved data manipulation, metric calculations, and deeper analytical exploration.

### 8. Data Visualization

Visual representations of the data were created to better understand patterns, relationships, and trends within the dataset. These visualizations help communicate insights effectively.

### 9. Hypothesis Testing

Statistical hypothesis testing was conducted to validate observed patterns and determine whether they were statistically significant.

### 10. Insight Generation

Key insights were extracted from the analysis, highlighting important trends, patterns, and relationships that could inform business decisions.

### 11. Final Report

All findings, visualizations, and insights were compiled into a final analytical report summarizing the outcomes of the project and providing actionable conclusions.
